Тестирование API для выдачи регионов
API https://regions-test.2gis.com/1.0/regions
1. нечеткий поиск
-  get-запрос GET/1.0/regions?q=Новосибирск
-проверяем минимальное количество символов в запросе
-проверяем максимальное количество символов в запросе
- задаем минимальное количество символов, например (нов) и смотрим не выдает ли 2 результата вместо одного
- если передан этот параметр остальные игнорируются, смотрим так ли это
2. параметр- код страны для фильтрации
- get-запрос GET/1.0/regions?country_code=ru
- get-запрос GET/1.0/regions?country_code=kg
- get-запрос GET/1.0/regions?country_code=kz
- get-запрос GET/1.0/regions?country_code=cz
- смотрим все ли коды стран указаны
3.Порядковый номер страниц
-минимальное значение страниц
-get-запрос GET/1.0/regions?page=1
-get-запрос GET/1.0/regions?page=2
-get-запрос GET/1.0/regions?page=3
4. Количество элементов на странице
- GET/1.0/regions?page=1&page_size=5 
- GET/1.0/regions?page=1&page_size=10      проверяем какое количество элементов может быть на первой странице
- GET/1.0/regions?page=1&page_size=15
-меняем параметр количества элементов на странице на 4,6,9,11,14,16
